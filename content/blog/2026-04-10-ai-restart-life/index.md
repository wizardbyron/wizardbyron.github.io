>一个人，如果不耕作，就必须写作。
>
>——Martin A. Hansen

这是我十几年前在《C++的设计和演化》开篇看到的一句话，也是我最近一直在做的事情。

过去一周，我开始用 AI 用 AI 设计并开始适应新的作息时间表，能够让我有充足的睡眠恢复精力，恢复昼夜节律。

然后，我继续用 AI 填充作息时间表的细节，比如：

- 用 AI 设计每天的食谱。自己做饭来控制热量、血糖、盐。此外，还可以避免孩子对一些常见食物（鸡蛋、牛奶、大豆、小麦）的过敏。
- 用 AI 设计健身恢复计划。包括大肌群、心肺以及体态上的小毛病。
- 用 AI 整理自己日常的写作发布流程。在苹果的“备忘录”中写作，然后通过skill 生成博客和微信公众号内容。（排版和样式也是用 AI从别人的公众号上 “学”的）

目前我暂定每天工作 6 小时（上午和下午各3小时），然后用剩下的时间健身、阅读、思考、并重新启用纸笔进行记录，和电子屏幕保持一些距离。

我觉得经历长期加班/压力的人都需要一段时间来重启自己，恢复精力和对生活的掌控感。

我制作的一些 Skill 会放在 [Agent Skills](/skills/)里面，欢迎自行取用。

## 关于 AI 技术选型

我这个人怕麻烦，目前没有需要就暂时不科学上网。

`Agentic Coding` 这块我用 `OpenCode` + `国产 Coding Plan` +  `Ollama` 备用。

`国产 Coding Plan` 我目前在用 `Kimi Code Plan` 和 国际版 `GLM Coding Plan`，选择的依据仍然是[SWE-Bench](https://www.swebench.com) 的排名。

**这里需要吐槽一下 GLM**。一方面 GLM Coding Plan国际版在5.1 升级后非常不稳定。其次是春节期间GLM国内搞”限时限量购”，但是海外不受限制。加上人民币升值的大背景，海外版理论上比国内便宜。所以我用自己的 Github 账号注册，没想到最近 Github 创建的帐号也要实名认证（估计是检测到我的 IP 是大陆的），认证后就完全找不到我之前填写的支付信息了。z.ai 的各种客服都不回复，还好我备份了 api-key。

目前用 AI 解决的问题还都没有到深水区，如果 Coding Plan 限流了就刚好休息一下，让 Ollama 来接手。

小龙虾方面，还是选择了官方的 `OpenClaw` + `微信ClawBot` + `飞书机器人`的形式，适用范围会比较广。

另外，可以选择WorkBuddy 和 QClaw 打辅助。桌面端的产品设计仍然是腾讯的主战场，GLM 模仿还要加把劲（无数槽点按下不表……），体验非常好。

此外腾讯文档也有了 MCP，这就可以发挥了更大的想象力。

作为腾讯的员工，还是有一些腾讯产品情结的。
